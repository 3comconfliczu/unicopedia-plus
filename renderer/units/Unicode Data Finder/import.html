Instructions
  • The Find by Name feature of the Unicode Data Finder utility displays a list of basic data (character, code point, name/aliases, block) of matching Unicode characters searched by name or alias, including through regular expressions.
  • After entering a query, click on the Search button to display a list of all relevant matches, if any, ordered by code point value.
  • It is possible to choose how many characters are shown one page at a time.
  • When available, name aliases are displayed (in italics and smaller typeface) after the unique and immutable Unicode name. A correction alias is indicated by a leading reference mark ※.
  • All names and aliases are obtained from the UnicodeData.txt and NameAliases.txt data files.
  • The search is performed on the 283,440 assigned characters (or code points) defined in the Unicode 13.0 version of the UnicodeData.txt data file.
  • Use the Results ▾ pop-up menu to perform an action among:
    • Copy Results [copy the results as string to the clipboard]
    • Save Results.. [save the results as string to a text file]
    • Clear Results [clear the current list of results]
  • Various examples of regular expressions are provided for quick copy-and-paste.
Regex Examples
  • Characters whose name starts with either "cjk radical" or "kangxi radical":
    ^(cjk|kangxi) radical
  • Characters whose name contains both "hiragana" and "katakana":
    ^(?=.*hiragana)(?=.*katakana)
  • Characters whose name contains both "katakana" and whole-word "a":
    ^(?=.*katakana)(?=.*\ba\b)
  • Characters whose name contains "bamboo" but not "radical":
    ^(?=.*bamboo)(?!.*radical)
  • Characters whose name contains words ending with "boat", but are not "boat":
    \b\w+boat\b
  • Characters whose name's length is greater than or equal to 72:
    .{72,}

Instructions
  • The Match Character feature of the Unicode Data Finder utility displays a list of basic data (character, code point, name/aliases, block) of Unicode characters matching a character, including through regular expressions.
  • After entering a query, click on the Search button to display a list of all relevant matches, if any, ordered by code point value.
  • Click on the (Match Decomposition) toggle button to extend the search to characters whose decomposition mapping matches the query string.
  • It is possible to choose how many characters are shown one page at a time.
  • The search is performed on the 283,440 assigned characters (or code points) defined in the Unicode 13.0 version of the UnicodeData.txt data file.
  • Use the Results ▾ pop-up menu to perform an action among:
    • Copy Results [copy the results as string to the clipboard]
    • Save Results.. [save the results as string to a text file]
    • Clear Results [clear the current list of results]
  • Various examples of regular expressions are provided for quick copy-and-paste.
Regex Examples

(Match Decomposition) toggled off:

  • Characters with the 'White_Space' binary property:
    \p{White_Space}
  • Characters considered white space in Unicode by the regular expression engine:
    \s
  • Characters considered line terminators in Unicode by the regular expression engine:
    (?!.)[^] or (?!.)[\s\S]
  • All assigned Unicode characters:
    .* or [^] or [\s\S]
  • Characters with the 'General_Category' property equal to 'Private_Use':
    \p{Private_Use}
  • Characters without the 'General_Category' property equal to 'Private_Use':
    \P{Private_Use} or [^\p{Private_Use}]
  • Characters without the 'General_Category' property equal to 'Control', 'Surrogate' or 'Private_Use', i.e., the total of 143,859 Unicode characters defined in Unicode 13.0:
    [^\p{Control}\p{Surrogate}\p{Private_Use}]
    or
    (?=\P{Control})(?=\P{Surrogate})(?=\P{Private_Use})
  • Characters with the 'General_Category' property equal to 'Open_Punctuation' or 'Close_Punctuation':
    [\p{Open_Punctuation}\p{Close_Punctuation}]
    or
    \p{Open_Punctuation}|\p{Close_Punctuation}
  • Characters with the 'Dash' binary property, but without the 'General_Category' property equal to 'Dash_Punctuation':
    (?=\p{Dash})(?!\p{Dash_Punctuation})
    or
    (?=\p{Dash})(?=\P{Dash_Punctuation})
  • Characters with the 'Radical' binary property:
    \p{Radical}
  • Characters with the 'Hex_Digit' binary property:
    \p{Hex_Digit}
  • Characters with the 'Regional_Indicator' binary property:
    \p{Regional_Indicator}
  • Characters with the 'Emoji_Modifier' binary property:
    \p{Emoji_Modifier}
  • Characters with the 'Script' property equal to 'Hiragana':
    \p{Script=Hiragana}
  • Characters belonging to the Unihan set, i.e., with the 'Script' property equal to 'Han' and the 'General_Category' property equal to 'Other_Letter':
    (?=\p{Script=Han})(?=\p{Other_Letter})
  • Characters with the 'Script' property equal to 'Latin' and the 'General_Category' property equal to 'Number':
    (?=\p{Script=Latin})(?=\p{Number})
  • Characters with the 'Emoji_Modifier_Base' binary property, but without the 'Emoji_Presentation' binary property:
    (?=\p{Emoji_Modifier_Base})(?!\p{Emoji_Presentation})
    or
    (?=\p{Emoji_Modifier_Base})(?=\P{Emoji_Presentation})
  • Characters with the 'Ideographic' binary property, but without the 'Unified_Ideograph' binary property:
    (?=\p{Ideographic})(?!\p{Unified_Ideograph})
    or
    (?=\p{Ideographic})(?=\P{Unified_Ideograph})
  • Characters with the 'Script_Extensions' property equal to 'Han', but without the 'Script' property equal to 'Han':
    (?=\p{Script_Extensions=Han})(?!\p{Script=Han})
    or
    (?=\p{Script_Extensions=Han})(?=\P{Script=Han})

(Match Decomposition) toggled on:

  • Character made of four integrals, i.e., a quadruple integral operator:
    ∫{4}
  • Characters containing two digits (0 to 9):
    \d\d
  • Characters made of two letters separated by a slash:
    \w/\w
  • Characters which are latin small ligatures starting wih 'f':
    f[fil]
  • Characters made of a single katakana letter 'ho':
    ^ホ$
  • Characters containing either 上 (up, top, high) or 下 (down, bottom, low):
    上|下
  • Characters containing 左 (left), 中 (middle) or 右 (right):
    左|中|右
  • Characters containing one ideograph, Japanese shorthand for each of the seven days of the week:
    [月火水木金土日]
  • Characters made of a single bracketed ideograph:
    〔\p{Script=Han}〕
  • Characters made of a single parenthesized ideograph:
    \(\p{Script=Han}\)
  • Characters made of Arabic letters separated by white space, forming intricate Arabic ligatures:
    \p{Script=Arabic}\s\p{Script=Arabic}

Instructions
  • The List by Block feature of the Unicode Data Finder utility displays in real time a list of basic data (character, code point, name/aliases, block) of Unicode characters belonging to the same block range.
  • It is possible to choose how many characters are shown one page at a time.
  • A block can be selected either by Block Name or by Block Range, as defined in the Blocks.txt data file.
  • It is also possible to directly enter a code point (or character) in the Specimen field, then click on the Go button to automatically select the block containing the code point, scroll its basic data into view, and highlight its hexadecimal code value.
  • You can quickly reuse a previously entered code point by using the Alt+↑ and Alt+↓ keyboard shortcuts to navigate up and down through the history stack in the Specimen field. Alternatively, you can also use the Specimen History ▾ pop-up menu to automatically jump to a specific code point.
  • Use the Results ▾ pop-up menu to perform an action among:
    • Copy Results [copy the results as string to the clipboard]
    • Save Results.. [save the results as string to a text file]
